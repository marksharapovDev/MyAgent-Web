# Telegram-бот — UX, команды, сценарии

## Состояния бота (State Machine)

Бот всегда находится в одном из пяти состояний.
Состояние хранится в Redis: `bot:state:{user_id}`.

```
idle → discussing → executing → reviewing → idle
          │              ↑           ↓
          │              └───────────┘ (правки)
          │
          └→ planning → autopilot ←→ detail
                              ↓
                          executing → reviewing → (следующая задача)
```

| Состояние | Что происходит | Что принимает |
|-----------|---------------|---------------|
| **idle** | Ждёт задачу. Показывает активный проект | Текст, голос, фото, команды |
| **discussing** | Обсуждение деталей с Claude | Текст, голос, кнопки, /cancel |
| **planning** | Создание плана проекта | Текст, голос, кнопки |
| **autopilot** | Агент работает по плану, редкие чекины | /detail, /pause, /status, /plan |
| **executing** | Агент выполняет одну задачу | Только /cancel и /status |
| **reviewing** | Показан результат, ждёт реакцию | Текст (правки), кнопки, голос |

## Команды

### /start
Первый запуск. Проверяет TELEGRAM_ALLOWED_USERS.

```
Бот: "Привет! Я твой агент для веб-разработки.

      Для начала работы создай первый проект
      или выбери существующий."

      [➕ Создать проект] [📂 Мои проекты]
```

### /project
Выбор активного проекта. Все последующие сообщения
привязываются к выбранному проекту.

```
Бот: "Выбери проект:"

      [🟢 Хлеб&Соль (активный)]
      [⚪ Автосервис Молния]
      [⚪ Фитнес-клуб Олимп]
      [➕ Создать новый]
```

После выбора:
```
Бот: "Переключился на «Хлеб&Соль»
      Статус: 3 страницы готовы, деплой на Vercel ✅

      Что делаем?"
```

### /new
Создание нового проекта. Запускает мини-диалог для сбора базовой информации.

```
Бот: "Создаём новый проект. Как он называется?"

Ты:  "Автосервис Молния"

Бот: "Отлично. Опиши в двух словах что нужно сделать?"

Ты:  🎤 "Лендинг для автосервиса, нужна запись на обслуживание,
      прайс-лист и отзывы"

Бот: "Понял. Стек по умолчанию — Next.js + Tailwind + Supabase.
      Подходит или нужно что-то другое?"

      [✅ Подходит] [⚙️ Изменить стек]

Ты:  ✅

Бот: "Проект «Автосервис Молния» создан!
      🆔 slug: avtoservis-molniya
      📋 Бриф сохранён

      Начинаем работу? Что делаем первым?"
```

### /status
Показывает текущее состояние: активные задачи, последний деплой, прогресс.

```
Бот: "📊 Статус «Хлеб&Соль»

      ✅ Главная страница — готова, задеплоена
      ✅ О нас — готова, задеплоена
      🔄 Контакты — дизайн v2, ждёт правки
      ⏳ Каталог — в очереди

      Последний деплой: 2 часа назад (Vercel)
      Preview: https://hleb-sol-xxx.vercel.app"
```

### /deploy
Быстрый деплой текущего проекта.

```
Бот: "Куда деплоим «Хлеб&Соль»?"

      [🔵 Vercel (preview)] [🔴 VPS (production)]

Ты:  🔵 Vercel

Бот: "Деплою на Vercel из ветки main..."
      ⏳ Сборка...
      ⏳ Загрузка...
      ✅ Готово!

      🔗 https://hleb-sol-xxx.vercel.app

      [📤 Отправить ссылку клиенту] [🔴 Деплой на прод]
```

Для production:
```
Ты:  🔴 VPS

Бот: "⚠️ Деплой на продакшен «Хлеб&Соль»
      Сервер: 185.xxx.xxx.xx
      Домен: hlebisol.ru

      Это НЕОБРАТИМО. Текущая версия будет заменена.
      Бэкап будет создан автоматически.

      Подтверждаешь?"

      [✅ Да, деплой на прод] [❌ Отмена]
```

### /docs
Управление документацией проекта.

```
Бот: "📄 Документация «Хлеб&Соль»

      1. Бриф (обновлён 2 дня назад)
      2. Стилегайд (обновлён 5 дней назад)
      3. Архитектура (обновлён 1 день назад)

      Что сделать?"

      [👁 Посмотреть] [✏️ Обновить] [➕ Добавить новый]
```

При обновлении:
```
Ты:  ✏️ → Стилегайд

Бот: "Текущий стилегайд:
      • Шрифт: Inter
      • Цвета: #2D1810 (коричневый), #F5F0EB (беж)
      • Стиль: тёплый, уютный, крафтовый

      Отправь изменения текстом или голосом."

Ты:  🎤 "Добавь ещё акцентный цвет — тёмно-зелёный,
      примерно как #1B4332. И шрифт для заголовков
      поменяй на Playfair Display"

Бот: "Обновил стилегайд:
      • Шрифт заголовков: Playfair Display (было: Inter)
      • Шрифт текста: Inter (без изменений)
      • Добавлен акцентный цвет: #1B4332
      ✅ Сохранено"
```

### /history
Последние выполненные задачи.

```
Бот: "📜 Последние задачи «Хлеб&Соль»

      1. 🎨 Дизайн контактов v2 — 30 мин назад (reviewing)
      2. 💻 Форма обратной связи — 2 часа назад (done)
      3. 🚀 Деплой на Vercel — 3 часа назад (done)
      4. 🎨 Дизайн контактов v1 — вчера (rejected)
      5. 💻 Страница «О нас» — вчера (done)"
```

### /cancel
Отменяет текущее действие.

```
В состоянии discussing:
Бот: "Обсуждение отменено. Что дальше?"

В состоянии executing:
Бот: "⚠️ Задача «Дизайн контактов» сейчас выполняется.
      Отменить?"
      [Да, отменить] [Нет, пусть работает]

В состоянии reviewing:
Бот: "Результат отклонён. Начинаем заново или меняем задачу?"
      [🔄 Заново] [✏️ Изменить задачу] [🗑 Пропустить]
```

### /plan
Показывает текущий план проекта и позволяет изменить.

```
Бот: "📋 План «Хлеб&Соль» (v2)

      Этап 1: Дизайн
        ✅ 1.1 Главная (вариант A одобрен)
        ✅ 1.2 О нас
        🔄 1.3 Каталог ← сейчас
        ⏳ 1.4 Контакты
        ⏳ 1.5 Отзывы

      Этап 2: Разработка
        ⏳ 2.1 — 2.4 (ждёт дизайн)

      Этап 3: Деплой
        ⏳ 3.1 — 3.2

      Режим: 🤖 автопилот"

      [✏️ Изменить план] [🔍 Детальный] [⏸ Пауза]
```

### /autopilot
Переключает в режим автопилот.

```
Бот: "🤖 Переключился в автопилот.
      Продолжаю работу по плану, вернусь на чекпоинтах.

      Текущий этап: Дизайн → Каталог
      Следующий чекпоинт: после завершения всех страниц."
```

### /detail
Переключает в детальный режим.

```
Бот: "🔍 Переключился в детальный режим.
      Буду показывать каждый шаг.

      Сейчас работаю над: Каталог → сетка товаров.
      Показать текущий вариант?"
```

### /pause
Ставит автопилот на паузу.

```
Бот: "⏸ Автопилот на паузе.
      Текущая задача (Каталог) будет завершена.
      Новые задачи не начнутся.

      [▶️ Продолжить] [📋 План]"
```

### /resume
Возобновляет автопилот.

```
Бот: "▶️ Автопилот возобновлён.
      Продолжаю с задачи: Контакты."
```

### /help
```
Бот: "🤖 Команды:

      /project — выбрать проект
      /new — создать проект
      /status — статус задач и деплоев
      /deploy — деплой проекта
      /docs — документация проекта
      /history — история задач
      /plan — план проекта
      /autopilot — режим автопилот
      /detail — детальный режим
      /pause — пауза автопилота
      /resume — продолжить
      /cancel — отменить текущее

      Или просто напиши/скажи что нужно сделать."
```

## Основные сценарии

### Сценарий 1: Дизайн страницы (полный цикл)

```
Состояние: idle
Активный проект: Хлеб&Соль

Ты:  "Нужна страница с каталогом хлеба"

      → Состояние: discussing

Бот: "Каталог — отлично. Уточню пару моментов:
      1. Это просто витрина (фото + описание + цена)?
         Или с возможностью заказа?
      2. Сколько примерно позиций?"

Ты:  🎤 "Витрина, штук 12 позиций, разбей по категориям —
      хлеб, выпечка, торты"

Бот: "Понял. Получается так:

      📋 Каталог продукции
      • Три категории: хлеб, выпечка, торты
      • ~12 позиций с фото, названием, описанием и ценой
      • Фильтрация по категориям (табы или кнопки)
      • Стиль как на остальных страницах

      [✅ Всё верно, делай] [✏️ Ещё кое-что]"

Ты:  ✅

      → Состояние: executing

Бот: "⏳ Генерирую дизайн каталога..."

      (30-60 секунд)

Бот: [скриншот desktop версии]
      [скриншот mobile версии]

      "Вот что получилось. Использовал карточки
      с тенью, табы для категорий."

      → Состояние: reviewing

      [✅ Отлично] [✏️ Правки] [🔄 Переделать с нуля]

Ты:  ✏️

Бот: "Опиши что изменить."

Ты:  "Карточки слишком широкие, сделай 3 в ряд на десктопе,
      и цену сделай крупнее и жирнее"

      → Состояние: executing

Бот: "⏳ Вношу правки..."

      (20-30 секунд — быстрее, т.к. правки, не с нуля)

Бот: [обновлённый скриншот]

      "Готово: 3 колонки, цена увеличена и bold."

      → Состояние: reviewing

      [✅ Отлично] [✏️ Ещё правки] [🔄 Переделать]

Ты:  ✅

Бот: "Дизайн каталога одобрен (v2) ✅
      Код сохранён в Git: feature/catalog-page

      Что дальше?"

      → Состояние: idle

      [🚀 Задеплоить] [💻 Подключить данные из Supabase]
```

## Обработка ошибок

### Claude API недоступен
```
Бот: "⚠️ Не могу подключиться к AI. Повторю через 30 секунд.

      [🔄 Повторить сейчас] [⏭ Отмена]"
```

### Задача слишком долго выполняется (>2 мин)
```
Бот: "⏳ Задача выполняется дольше обычного...
      Статус: генерация кода (60%)

      [⏭ Продолжать ждать] [❌ Отменить]"
```

### Ошибка деплоя
```
Бот: "❌ Деплой провалился:
      Error: Build failed — Cannot find module 'xxx'

      Варианты:
      [🔧 Попробовать починить автоматически]
      [📋 Показать полный лог]
      [↩️ Откатить]"
```

### Неизвестный проект
```
Ты:  "Обнови главную страницу"
Бот: "У тебя нет активного проекта. Выбери:"
      [список проектов]
```

## Технические детали

### Сессии
- Каждый диалог привязан к user_id + project_id
- Кэш обсуждения: Redis с TTL 24 часа
- Состояние бота: Redis с TTL 1 час (auto-reset в idle)

### Throttling
- Не более 1 executing задачи одновременно
- Если отправляешь новую задачу во время executing:
  "⏳ Задача «Дизайн каталога» ещё выполняется.
   Подождать или отменить?"

### Безопасность
- TELEGRAM_ALLOWED_USERS — whitelist из .env
- Любое сообщение от другого user_id → игнор (без ответа)
- Все команды требуют активного проекта (кроме /new, /start, /help)

### Файлы и медиа
- Фото → сохраняются в Supabase Storage, URL передаётся в промпт
- Голосовые → конвертация OGG → WAV → Deepgram → текст
- Документы (PDF) → текст извлекается, добавляется в context_docs
